# Common Ground

**Editorial operating system for building better decisions through common ground.**

Common Ground is an editorial publishing system for turning everyday friction into practical, reusable decision criteria.

It does not build trust around a personality. It builds trust through clear problems, observable evidence, consistent criteria, and useful outcomes.

## Brand

- Channel: **Common Ground**
- Editor: **Editor Zero**
- Core question: **Can this resolve a real inconvenience?**
- Principle: **Who said it matters less than whether the information works.**

## Repository Areas

- `docs/brand/` — channel DNA, Editor Zero identity, tone and voice
- `docs/pm/` — PM role, onboarding, operating rules, review criteria
- `docs/editorial/` — research, briefing, drafting, review, and update workflow
- `docs/seo/` — search intent, discoverability, and measurement guidance
- `docs/publishing/` — Naver Blog and future publishing operations
- `docs/templates/` — reusable briefs, article structures, and review forms
- `knowledge/` — verified platform knowledge and accumulated operating evidence
- `workflows/` — executable operating procedures and lifecycle definitions
- `prompts/` — approved generation and review prompts
- `assets/` — brand and publishing assets
